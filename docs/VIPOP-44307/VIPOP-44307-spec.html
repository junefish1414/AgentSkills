4. 操作流程

流程一：點擊詢問意願 / 邀約測評（廣告暫停/關閉時）

現行流程：廣告暫停/關閉 → 詢問意願/邀約測評按鈕不顯示，用戶無法知道功能存在，誤以為系統異常。

調整後流程：按鈕顯示但 disable，按鈕區域上方顯示提示 banner，提供 [開啟廣告] CTA 另開分頁。

flowchart TD
    A[用戶進入 ATS 甄試履歷頁面] --> B{廣告狀態}
    B -->|刊登中| C[詢問意願/邀約測評 正常可用]
    B -->|暫停中 或 關閉中| D[詢問意願/邀約測評 顯示但 disable]
    D --> E[上方 banner：廣告尚未開啟，開啟後即可使用完整功能]
    E --> F{用戶點擊 開啟廣告}
    F -->|暫停中| G["另開新分頁\n/rms/ad/status?isShowReOpenModal=1\n直接顯示立即恢復刊登圖層"]
    F -->|關閉中| H["另開新分頁\n/rms/order/general/selectPlan"]

流程二：新增行程（三步驟圖層，廣告暫停/關閉時）

現行流程：Step 2 被跳過，直接預設「不通知求職者」。

調整後流程（2026/04/27）：Step 2 回歸顯示，但通知求職者選項鎖定為「不通知」且不可修改；上方 banner 說明廣告未開啟；下方提示＋[開啟廣告] CTA。

flowchart TD
    A["用戶觸發邀約動作\n(拖曳/移動至/行事曆新增)"] --> B{廣告狀態}
    B -->|刊登中| C[Step 1 → Step 2 正常流程 → Step 3]
    B -->|暫停中 或 關閉中| D["Step 1：填寫行程資訊\naction btn: 新增"]
    D --> E["Step 2：回歸顯示但鎖定\n頂部 banner: 廣告未開啟不提供聯絡\n通知求職者=不通知且鎖定\n下方提示+開啟廣告CTA"]
    E --> F{用戶選擇}
    F -->|點擊開啟廣告| G[另開分頁處理廣告]
    F -->|繼續完成| H["Step 3：完成\n顯示通知圖示+廣告未開啟提示"]

流程三：修改行程（廣告暫停/關閉時）

flowchart TD
    A[用戶點擊修改行程] --> B{廣告狀態}
    B -->|刊登中| C[正常修改流程 → Toast 完成提示]
    B -->|暫停中 或 關閉中| D["修改畫面內顯示醒目提示\n本次異動不會通知求職者＋開啟廣告CTA"]
    D --> E[用戶完成修改送出]
    E --> F["Modal 彈窗\n標題: 修改完成，但未通知求職者\n內文: 本次異動不會通知求職者"]
    F --> G[用戶點擊確認關閉]

流程四：刪除 / 退回未過期行程（廣告暫停/關閉時）

flowchart TD
    A["用戶操作刪除行程\n或退回/移動至"] --> B{行程是否過期}
    B -->|已過期| C[正常操作，無需提示]
    B -->|未過期| D{廣告狀態}
    D -->|刊登中| E[正常流程，發 BC 溝通]
    D -->|暫停中 或 關閉中| F["確認 pop-up 內\n新增醒目底色提示\n本次異動不會通知求職者"]
    F --> G[用戶確認]
    G --> H[執行操作，不發 BC 溝通]

流程五：取消錄用（到職日未過期，廣告暫停/關閉時）

flowchart TD
    A[用戶操作取消錄用] --> B{到職日是否過期}
    B -->|已過期| C[正常取消，無需提示]
    B -->|未過期| D{廣告狀態}
    D -->|刊登中| E[正常流程，發 BC 溝通]
    D -->|暫停中 或 關閉中| F["pop-up 新增醒目提示\n＋開啟廣告CTA"]
    F --> G[用戶確認]
    G --> H[執行取消，不發 BC 溝通]
